Configure

Set your BigQuery source table and run metadata. These values appear in the results output.

☁️ BigQuery Connection
🏷️ Run Metadata
These values appear as static columns in every row of the results output — useful for tracking runs across months and systems.

Define Query

Write the SELECT statement and optional WHERE filters. Records excluded by the WHERE clause are counted as Filtered in results.

SELECT Statement
Write the full SELECT including FROM and any JOINs. Do not add a WHERE clause here — use the filter box below.
BigQuery SQL
WHERE Filters — optional, excludes records before DQ checks
WHERE clause only — no WHERE keyword
Tally will run SELECT COUNT(*) FROM (your SELECT) to get total records, then apply this WHERE clause and count evaluated records. The difference is your Filtered count.
JOIN Another Table — optional

Select PDEs & Grouping

Load table columns, choose which Physical Data Elements to check, and set the grouping column for results.

Physical Data Elements
None selected
Click ⟳ Load Columns to fetch from BigQuery.
Results Grouping
Results will show one row per (group value × PDE). Typically your Line of Business column.
Static Output Columns
These appear as fixed columns on every result row — useful for filtering in Qlik / reports.

Define Rules

Configure data quality rules for each selected PDE. Rules cascade — Conformity runs on Completeness pass, Specificity on Conformity pass, Reference on Specificity pass.

Completeness — always applied
Conformity — on completeness pass
Specificity — on conformity pass
Reference — on specificity pass
⚙️
Go back to Step 3 and select PDEs first.

Run & Results

Execute all DQ checks. One BigQuery query runs per PDE — results are combined into a single summary table.

Execution
Tally will run: 2 + N queries — 1 for total count, 1 for evaluated count, then one per PDE for rule checks. Large tables may take a few minutes.

Click Run DQM to execute all checks.
🎫 Raise Jira Ticket — notify the data team of DQ exceptions

Create a Jira ticket with the DQ summary. The data team can review and reply with explanations or remediation actions. The AI agent reads their replies and provides recommendations.

AI Agent

Once the team replies on Jira, click Ask Agent on the ticket card. Claude will read the comments and provide a data quality assessment with root cause analysis and remediation recommendations.
